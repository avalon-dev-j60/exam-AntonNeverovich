# exam
### Аттестационное задание по курсу "DEV-J60. Технологии разработки корпоративных приложений на платформе Java Enterprise Edition (Java EE)"
***
Разработать сайт с использованием технологий редакции Java Enterprise Edition.

Тематическая направленность сайта может быть выбрана слушателем произвольно. Например, сайт любителей рыбалки, или новостной ресурс в области среднего образования.

Для успешной аттестации должны выполняться следующие требования:
***
**Нефункциональные требования:**

REQ-001: Все динамические данные должны храниться в базе данных.

REQ-002: Взаимодействие с базой данных должно быть реализовано с использованием Java Persistence API (JPA).

REQ-003: Все сущностные классы и сеансовые компоненты (компоненты уровня предприятия) должны быть расположены в EJB-модуле.

REQ-004: Приложение должно иметь веб-интерфейс.

REQ-005: Представление веб-приложения должно быть реализовано с использованием технологии Java Server Pages (JSP).

REQ-006: Все HTTP-запросы должны обрабатываться с использованием технологии Java Servlets.

***
**Функциональные требования:**

REQ-007: При окончании пользовательского сеанса система требует повторной аутентификации.

REQ-008: Любой пользователь может свободно зарегистрироваться в системе.

REQ-009: Пользователь может редактировать только то динамическое содержимое, автором которого является.

REQ-010: Редактирование динамического содержимого требует авторизации.

REQ-011: Все пользователи сайта могут просматривать всё динамическое содержимое системы без авторизации.

***
Код решения следует загрузить в репозиторий. Создать репозиторий на портале GitHub можно по предложенной ссылке.

Проверка степени готовности аттестационного задания будет осуществляться в урочное время в начале каждого практического занятия. Для упрощения проверки и обеспечения готовности задания к последнему практическому занятию, предлагается разбить задание на следующие части:

1. оформление элементов пользовательского интерфейса с использованием технологий JSP, HTML и CSS.

2. описать предметную область (сущностные классы) и сеансовые компоненты для сущностных классов;

3. описать сервлеты и бизнесс-логику.
